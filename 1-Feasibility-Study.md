## 2. Expected Revenue Potential

Assume the app takes a **10% commission** per booking, with average booking value of **$500**.

- 1,000 users/month × $500 × 10% = **$50,000/month**
- Yearly projection: $50,000 × 12 = **$600,000/year**

## Technical Feasibility

The system is technically feasible based on current mobile/web technologies. The design demonstrates:

- Account management (Sign up / Log in / Profile)
- Multi-city booking with calendar integration
- Payment integration
- Flight tracking
- Mobile responsiveness

### Frontend Technologies

| Component | Technology | Notes |
|---|---|---|
| Framework | Flutter (Dart) | Recommended for cross-platform Android/iOS |
| UI Toolkit | Flutter Widgets / Material Design | Aligns with modern UI |
| Alternative | ReactJS (for web-only) | If separating web & mobile |
| Navigation | Flutter Navigator / GoRouter | Advanced routing |

### Backend Technologies

| Layer | Technology | Notes |
|---|---|---|
| Backend Framework | Node.js with Express.js | Fast, scalable, good for REST APIs |
| OR Alternative | Django (Python) / Laravel (PHP) | Strong admin support |
| Authentication | Firebase Auth / JWT | Secure built-in auth |
| Database | PostgreSQL / MongoDB | Relational or document store |
| Payment | Stripe / PayPal SDK | Proven payment gateways |

---

## Operations Feasibility

- The app is operationally feasible with minimal staff and cost-efficient systems.
- The design choices (account system, booking flow, clear UI) make it user-friendly and scalable.
- With planned backend architecture and customer support integration, the system can function efficiently from day one.
